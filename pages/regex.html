What is regex?

In a more simplistic description, regex is some fancy-looking symbols put together to find certain parts of a string.

Take, for example, the emergency care number below:

+27 999

+353 999

The second number is the correct Irish phone number, while the first is not. But how do we tell the difference?

Selecting the Irish phone number

Jumping straight in, let's tackle the Irish phone number.

For this, we want to use a numerical selector to select all the digits.

Regex

\d

Output

+353 999
+27 999

Huh? Well, that selected all the digits, it also selected the digits we don't want. (If you use Regex101 you will also see that all numbers are individually selected; more on this later).

We need a way to select the digits if they appear after +353.

Let's try adding +353 to the beginning of our expression.

Regex

+353\d

Output

+353 999
+27 999

Drats! We ran into another issue. The "+" symbol is already a modifier. We can stop any symbol from acting as a modifier by escaping. Use a backslash to escape any character. This will not work however. Recall our previous regex expression

Regex

+353\d

Escaping this will then allow us to select "+353", but then the "\d" all digits selector wants to select any digits after this number.

Regex will read it like this.

  • Find +353
  • Find (\d) any digits after that

This is a problem. Our Irish phone number has a space between "+353" and the number. To fix the issue we just add a space between "+353" and "\d".

Regex

\+353 \d

By breaking this down we can understand what we are trying to tell regex.

  • (\+353) tells regex to select all values that have "+353"
  • ( \d) tells regex to select a digit if the following condition is met plus this condition

Output

+353 999
+27 999

We're almost there. It's selecting the correct "row", but it's not selecting all the digits. This is because "\d" is non-repeating. The reason it selects all digits without the "\+353" is because the selection was vague (any digit matched the expression).

To fix this issue, we have two options. Use the star ("wildcard" selects 0 or more) modifier that selects everything, or use a plus modifier that selects everything (1 or more).

Try experimenting with both. For this guide I will use the plus modifier.

Regex

\+353 \d+

Output

+353 999
+27 999

Great Work! You've built your first regular expression.


Having A Look Around.

Let's present a problem. Suppose we want to select the domain of the URL.

https:// www.google.com /

https:// www.youtube.com /

randomProtocol:// www.introtowebdev.ie /

Take some time to try and come up with your best solution, then continue to read the rest of the guide.

You would notice that it's hard to find a universal pattern that selects just the domain of the URL. It would be convenient to just tell regex to select everything after the protocol, but stop selecting before the path.

Luckly, Javascript Regex implements "looking" functions. Note that not all regex modules are built equal and some languages have support for advanced features like "looking around".

There 4 expressions we are interested in are:

Expression Meaning
(?=) Positive Lookahead
(?<=) Positive Lookbehind
(?!) Negative Lookahead
(?<!) Negative Lookbehind

The "look around" expressions function quite similarly to if statements. Let's break down the positive lookahead:

  • ()

    The brackets are called a "group selector". This will separate anything within the brackets into it's own match group (more on this later).

  • ?=

    The question mark and equals sign represent a query selector.

The positive lookahead expression will put an invisible marker before any matches (this also depends on whether the statement is greedy or not; more on this later).

Match example, (?=)

|match before me

Likewise, the positive lookbehind functions the same as the positive lookahead, except it puts an invisible marker after any matches.

The '<' is used to decide which side the cursor should be placed.

Match example, (?<=)

match after me|

Negative "look-arounds" are used as the "else" part of an if statement. IF this expression does not match the negative "look-around" then select it. Negative lookarounds select anything that does not match the expression.

How to solve to problem

We need to select everything after the '://' but before the ending '/'.

Match example

://www.google.com/

To do this we use a lookbehind and a lookahead.

Make sure to escape the forward-slash by using a backslash.

Regex, Lookbehind

(?<=:\/\/)

Regex, Lookahead

(?<=?\/)

Regex, Lookbehind + Everything + Lookahead

(?<=:\/\/).*(?=\/)

Breaking the expression down:

  • (?<=:\/\/)

    Put a cursor after '://'

  • .*

    Wildcard selector that will select everything after it

  • (?=\/)

    Put a cursor before '/'

What this results in, is that the wildcard selector is nested between two endpoints. This gives us our desired result, selecting just the domain. Not bad!

Regex

(?<=:\/\/).*(?=\/)

Test String

https://www.google.com/
https://www.youtube.com/
randomProtocol://www.introtowebdev.ie/

Output

https://www.google.com/
https://www.youtube.com/
randomProtocol://www.introtowebdev.ie/

Grouping and Combinations

How to group

In regular-expressions, grouping is the act in which we can sort information. This group is called a 'capture group'.

Unfortunately, my regex editor does not display different groups. I highly recommend using Regex 101 to see different groups.

Try the following regex expression and notice how 'a' and 'b' get split into group 0, and 1 respectively.

Regex

a|(b)

Test String

ab

If there is no specified capture group, the default group is 0.

You may have also noticed we used a pipe '|'. A pipe is used as an 'or' operator in regex.

We can use the pipe to combine different selectors together. Recall that I mentioned not all languages support regex equally, while some languages, like Perl, will support more statements and expressions than just the 'or' operator. Be sure to check if your language supports the regex feature before trying it.

Regex

a|(b)

Test String

ab

Another 'grouping' method is the square brackets. The square brackets aren't actual groups, as they perform individual selection.

[]

The square brackets will not group together a continuous string. It will invidiually select everything within them. Try it out.

Regex

[abc]

Test String

Hello. A regex expression is a collection of symbols used to select or query a string.

Output

Hello. A regex expression is a collection of symbols used to select or query a string.

Here's a practice problem. We want to select the trailing white spaces, '\n', at the front and end of the string only, and then split these into two groups; 1 and 2.

Practice Problem

\n<div>\n\t<p>\n\t\tHello World, from the index.html file.\n\t</p>\n</div>\n

Regex

(^\\n)|(\\n$)

Explanation:

  • (^\\n)

    Selects everything at the start of the string that has a backslash, '\', and a 'n'

  • |

    OR operator that combines both capture groups together

  • (\\n$)

    Selects everything at the end of the string that has a backslash, '\', and a 'n'

Negative Selecting and Non-Greedy Modifiers

One of the final topics I will cover in this guide is negative selecting and non-greedy(reluctant) modifiers.

We'll start of with negative selecting. It's pretty straightforward, with a few caveats.

Regex

[^]

The negative selector will select everything that is not between the square brackets.

The keen-eyed among might be asking, "Why do we need a negative selector when we have a negative lookaround?"

That's because I jumped the gun. Normally when learning regex you learn the very basics, a.k.a. negative selector. The issue with the negative selector is that it can only function as a non-grouping expression. I wanted to introduce you to lookarounds first as they allow you to build useful regex expressions without knowing a lot.

Here's an example of a negative selector:

Regex

[^abc]

Test String

Hello. A regex expression is a collection of symbols used to select or query a string.

Output

Hello. A regex expression is a collection of symbols used to select or query a string

If you use Regex 101 can see the non-grouping property of the negative selection.


Finally, we have the "?"

The non-greedy (lazy) modifier is the question mark.

?

The question mark means anything directly before it is optional (or it tells the engine to stop searching).

Here is an example to help understand the question mark:

Let's suppose we want to select all the text within double quotes. For simplicity sake we will include the quotes itself. If you want a challenge, try to select the text within the quotes without selecting the quotes itself.

Regex

".*"

Test String

"this is a value","this is another value"

Output

"this is a value","this is another value"

We can see that our expression is doing the right thing (sort of). The regex engine only detects the beginning and ending quotes; that's why everything in-between (including the quotes) is selected. It's not detecting that we have different double quotes. It's greedy. We need to make it non-greedy or lazy.

Regex

".*?"

Test String

"this is a value","this is another value"

Output

"this is a value","this is another value"

Here, we can see that it select individual quote groups. Grand!

Regex First Expression

([f]?\".*?\")|([f]?'.*?')|(.(?<=\([^f\"']).+?(?=\)))|\s+|\w+|\W
  • [f]?

    Capture-group that selects individual f's but is also optional

  • \".*?\"

    Selects everything within double quotes and is lazy. (The quotes are escaped to avoid errors in an IDE)

  • ([f]?'.*?')

    Exactly the same at the double quote selection only with single quotes

Regex Middle Expression

([f]?\".*?\")|([f]?'.*?')|(.(?<=\([^f\"']).+?(?=\)))|\s+|\w+|\W

This expression will selection everything within round brackets

  • .

    Will select the first letter of the character. This is because the cursor will be placed 1 letter ahead of the opening bracket

  • (?<=\([^f\"']).+?

    Select everything after the opening bracket if it's not a string

  • (?=\))

    Stop the selection before the closing bracket

Regex End Expression

([f]?\".*?\")|([f]?'.*?')|(.(?<=\([^f\"']).+?(?=\)))|\s+|\w+|\W
  • \s+

    Selects whitespaces and a plus modifier will select 1 or more of the whitespaces

  • \w+|\W

    Selects all characters [no spaces](Alphanumeric and non-alphanumeric)
